Introduction to Git

Version Control چیست؟

Version Control (کنترل نسخه) سیستمی است که تغییرات فایل‌ها را در طول زمان ثبت و مدیریت می‌کند. این سیستم به توسعه‌دهندگان کمک می‌کند:

  • نسخه‌های مختلف پروژه را نگهداری کنند
  • تاریخچه تغییرات را مشاهده کنند
  • در صورت نیاز به نسخه‌های قبلی بازگردند
  • به‌صورت تیمی روی پروژه کار کنند

چرا به Version Control نیاز داریم؟

بدون Version Control

  • تغییرات ممکن است از بین بروند
  • همکاری تیمی دشوار می‌شود
  • تاریخچه مشخصی از تغییرات وجود ندارد
  • بازگشت به نسخه‌های قبلی سخت است

با Version Control

  • تاریخچه کامل پروژه ذخیره می‌شود
  • تغییرات قابل پیگیری هستند
  • مدیریت نسخه‌ها ساده‌تر می‌شود
  • همکاری تیمی بهبود پیدا می‌کند

Git چیست؟

Git یک سیستم مدیریت نسخه (Version Control System) است که به شما کمک می‌کند تغییرات پروژه خود را مدیریت و ذخیره کنید.

Git می‌تواند برای مدیریت تغییرات در موارد زیر استفاده شود:

  • کدهای برنامه‌نویسی
  • فایل‌های متنی
  • پروژه‌های طراحی
  • مستندات و یادداشت‌ها

Git مانند ماشین زمان پروژه


    هر تغییری که در پروژه ایجاد می‌کنید ثبت می‌شود و می‌توانید در هر زمان به گذشته بازگردید و تغییرات را بررسی کنید.  
    این قابلیت باعث می‌شود پروژه ایمن‌تر و مدیریت تغییرات ساده‌تر شود.

ویژگی‌های اصلی Git

1. سیستم توزیع‌شده (Distributed)

Git یک سیستم توزیع‌شده است. یعنی هر فردی که روی پروژه کار می‌کند یک نسخه کامل از پروژه و تمام تاریخچه تغییرات را روی سیستم خود دارد.

مزایا

  • هر کاربر آرشیو کامل پروژه را دارد
  • امکان کار بدون اینترنت وجود دارد
  • وابستگی به سرور مرکزی کاهش می‌یابد
  • امنیت و سرعت بیشتر می‌شود

2. مستقل از اینترنت

Git برای انجام عملیات اصلی به اینترنت نیاز ندارد.

می‌توانید:

  • تغییرات را ثبت کنید
  • تاریخچه را مشاهده کنید
  • نسخه‌های مختلف پروژه را مدیریت کنید

بدون اتصال به اینترنت.

3. تمرکز بر تغییرات (نه فقط فایل‌ها)

در روش‌های سنتی، کل فایل ذخیره می‌شود. اما Git تغییرات بین نسخه‌ها را ثبت می‌کند.

Git مشخص می‌کند:

  • چه چیزی تغییر کرده
  • چه خطوطی اضافه شده‌اند
  • چه بخش‌هایی حذف یا ویرایش شده‌اند

مزایا

  • حجم پروژه کمتر می‌شود
  • سرعت بیشتر است
  • تاریخچه دقیق‌تر و شفاف‌تر است

4. مستقل از زبان برنامه‌نویسی

Git به زبان خاصی وابسته نیست و می‌تواند برای موارد زیر استفاده شود:

  • پروژه‌های برنامه‌نویسی
  • فایل‌های متنی
  • طراحی
  • مستندات

5. Git یک ابزار است، نه پلتفرم

Tool (ابزار)

  • روی سیستم نصب می‌شود
  • برای انجام یک کار مشخص استفاده می‌شود
  • از طریق ترمینال اجرا می‌شود

Platform (پلتفرم)

  • سرویس آنلاین است
  • محیط کامل با امکانات مختلف دارد
  • معمولاً دارای رابط گرافیکی و حساب کاربری است

Warning
Git یک ابزار مدیریت نسخه است و فضای میزبانی آنلاین پروژه ندارد.

تفاوت Git و GitHub

Git GitHub
ابزار مدیریت نسخه پلتفرم آنلاین
روی سیستم محلی اجرا می‌شود سرویس میزبانی پروژه
مدیریت تغییرات پروژه اشتراک‌گذاری و همکاری تیمی
آفلاین کار می‌کند نیاز به اینترنت دارد

خلاصه:

  • Git = ابزار مدیریت نسخه
  • GitHub = سرویس میزبانی پروژه

جمع‌بندی

در این بخش یاد گرفتیم:

  • Git یک سیستم مدیریت نسخه است
  • تاریخچه کامل تغییرات را نگهداری می‌کند
  • امکان بازگشت به نسخه‌های قبلی را فراهم می‌کند
  • همکاری تیمی را ساده‌تر می‌کند
  • Git یک ابزار است، نه پلتفرم
