05 — First Project

هدف این فصل

در این فصل یک پروژه واقعی را از صفر ایجاد می‌کنیم و چرخه‌ی کامل کار با Git را در عمل اجرا می‌کنیم.

هدف این بخش، اجرای عملی مفاهیم زیر است:

  • Working Directory
  • Repository
  • Staging Area
  • Commit
اینجا تمرکز بر «کار کردن» است، نه صرفاً اجرای دستورات.

سناریو

یک پروژه ساده با ساختار زیر ایجاد می‌کنیم:

my-project/
├── index.html
├── styles.css
└── README.md

این پروژه عمداً ساده است تا تمرکز فقط روی رفتار Git باشد.

مرحله 1 — ایجاد پروژه

mkdir my-project
cd my-project

در این لحظه فقط یک Working Directory داریم. هنوز هیچ Repositoryای وجود ندارد.

مرحله 2 — تبدیل به Repository

git init

اکنون Git این پوشه را به یک Repository محلی تبدیل کرده است.

ساختار .git ایجاد شده و Git آماده‌ی رهگیری تغییرات است.

مرحله 3 — ایجاد فایل‌ها

touch index.html styles.css README.md

بررسی وضعیت:

git status

فایل‌ها در وضعیت Untracked قرار دارند.

مرحله 4 — افزودن به Staging Area

. git add

بررسی مجدد:

git status

اکنون فایل‌ها در وضعیت Staged هستند.

مرحله 5 — ایجاد اولین Commit

git commit -m "Initial project structure"
  • یک Snapshot ثبت شده
  • تاریخچه آغاز شده
  • پروژه دارای نسخه است

بررسی تاریخچه:

git log

اولین Commit پروژه ثبت شده است.

چرخه کامل اجرا شد

در این فصل شما یک چرخه کامل را اجرا کردید:

Working Directory
→ Staging Area
→ Commit
→ History

این همان مدل عملیاتی Git است.

نتیجه

از این لحظه پروژه شما:

  • نسخه‌بندی می‌شود
  • قابل بازگشت است
  • دارای تاریخچه است
  • قابل توسعه ساخت‌یافته است
در فصل بعد وارد یکی از مهم‌ترین قابلیت‌های Git می‌شویم:

Branching
