08 — Remote Repositories

تا اینجا تمام کار ما محلی (Local) بود.

اما Git برای همکاری ساخته شده است.

Remote Repository نسخه‌ای از مخزن شماست که روی یک سرور قرار دارد و سایر افراد می‌توانند به آن دسترسی داشته باشند.

Remote چیست؟

Remote یک اشاره‌گر به یک Repository خارجی است.

این Repository می‌تواند روی:

  • GitHub
  • GitLab
  • Bitbucket
  • یا یک سرور خصوصی

قرار داشته باشد.

Remote فقط یک نام مستعار برای یک آدرس URL است.

origin دقیقاً چیست؟

وقتی یک پروژه را Clone می‌کنیم، Git به‌صورت پیش‌فرض Remoteای به نام origin ایجاد می‌کند.

origin نام خاص یا جادویی نیست.
فقط یک Convention است.

می‌توانست هر نام دیگری باشد.

مشاهده Remoteها:

git remote -v

افزودن Remote

git remote add origin https://github.com/user/project.git

اکنون Repository محلی به یک مخزن خارجی متصل شده است.

Push چیست؟

Push یعنی:

ارسال Commitهای محلی به Remote.

git push origin main

Git بررسی می‌کند که تاریخچه سازگار باشد، سپس تغییرات را منتقل می‌کند.

Pull چیست؟

Pull یعنی:

دریافت تغییرات از Remote و ادغام آن با Branch فعلی.

git pull origin main

در واقع:

Pull = Fetch + Merge

Fetch چیست؟

git fetch origin

Fetch تغییرات را دریافت می‌کند،
اما آن‌ها را وارد Branch فعلی نمی‌کند.

Fetch ایمن‌تر از Pull است چون ادغام خودکار انجام نمی‌دهد.

upstream چیست؟

وقتی برای اولین بار Push می‌کنید:

git push -u origin main

با -u یک رابطه Tracking ایجاد می‌شود.

از این پس می‌توانید فقط بنویسید:

git push
git pull

Branch محلی می‌داند به کدام Branch در Remote متصل است.

این اتصال را upstream می‌نامند.

Force Push

git push --force

یا:

git push -f

این دستور تاریخچه Remote را بازنویسی می‌کند.

خطرناک است.

اگر تیمی کار می‌کنید و دیگران Commit زده باشند، Force Push می‌تواند تاریخچه آن‌ها را نابود کند.

نسخه ایمن‌تر:

git push --force-with-lease

این دستور فقط در صورتی Force می‌کند که Remote تغییری نکرده باشد.

حذف Branch در Remote

git push origin --delete feature-login

Branch از سرور حذف می‌شود.

Clone چیست؟

git clone https://github.com/user/project.git

Clone:

  • Repository را دانلود می‌کند
  • Remote به نام origin می‌سازد
  • Branch پیش‌فرض را Checkout می‌کند

Clone ترکیبی از:

  • init
  • remote add
  • fetch
  • checkout

است.

مدلی از Remote

  • Local Repository → نسخه شخصی شما
  • Remote Repository → نسخه اشتراکی تیم
  • Push → ارسال تغییرات
  • Fetch → دریافت تغییرات
  • Pull → دریافت + ادغام

Remote هیچ چیز جادویی ندارد.
فقط یک Repository دیگر است که از طریق شبکه به آن متصل می‌شوید.

بهترین رویکرد حرفه‌ای

  • قبل از شروع کار: Pull یا Fetch
  • هنگام اتمام Push: Feature
  • هرگز Force Push روی Branch مشترک
  • Branchهای قدیمی را حذف کنید
  • از Pull Request برای Merge استفاده کنید (در پلتفرم‌های میزبانی)

نتیجه

اکنون می‌توانید:

  • Remote تعریف کنید
  • Push و Pull انجام دهید
  • Fetch را درک کنید
  • upstream تنظیم کنید
  • Force Push را ایمن مدیریت کنید
  • تاریخچه را بین Local و Remote هماهنگ کنید

در فصل بعد وارد موضوع همکاری تیمی و Workflowهای حرفه‌ای می‌شویم.
