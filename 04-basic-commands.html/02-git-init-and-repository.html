02 — Repository and git init

هدف این بخش

در بخش قبل با Working Directory آشنا شدیم و دیدیم که قبل از Git فقط با فایل‌ها و پوشه‌ها کار می‌کنیم و هیچ تاریخچه‌ای وجود ندارد.

در این بخش یاد می‌گیریم:

  • Repository چیست
  • دستور git init دقیقاً چه کاری انجام می‌دهد
  • پوشه مخفی .git چیست
  • چه زمانی یک پوشه به Repository تبدیل می‌شود

Repository چیست؟

Repository در ساده‌ترین تعریف یعنی: یک پوشه معمولی + ساختار داخلی Git برای نگهداری تاریخچه.

از دید سیستم‌عامل، هیچ تفاوتی بین یک پوش وجود ندارد؛ اما از دید Git، تفاوت بسیار مهمی ایجاد می‌شود.

هر پوشه‌ای که داخل آن پوشه‌ای به نام .git وجود داشته باشد، از نظر Git یک Repository محسوب می‌شود.

تبدیل یک پوشه به Repository

فرض کنید داخل پوشه git-practice هستید. این پوشه فعلاً فقط یک Working Directory ساده است.

اجرای دستور

git init

دستور git init چه می‌کند؟

وقتی دستور git init اجرا می‌شود:

  • یک پوشه مخفی به نام .git ساخته می‌شود.
  • ساختار داخلی لازم برای ذخیره تاریخچه ایجاد می‌شود.
  • Git از این لحظه شروع به مدیریت تغییرات این پوشه می‌کند.

اما نکته مهم:

  • هنوز هیچ فایلی ثبت نشده است.
  • هنوز هیچ Commit وجود ندارد.
  • هنوز هیچ Snapshot ساخته نشده است.

پوشه مخفی .git چیست؟

.git قلب Repository است و شامل موارد زیر می‌شود:

  • تاریخچه Commitها
  • Snapshotها
  • اشاره‌گرها (مثل HEAD)
  • تنظیمات داخلی Repository

اگر این پوشه حذف شود، کل تاریخچه پروژه از بین می‌رود. بنابراین هرگز نباید آن را به‌صورت دستی حذف یا ویرایش کرد (مگر اینکه دقیقاً بدانید چه می‌کنید).

تفاوت Working Directory و Repository

مفهوم توضیح
Working Directory پوشه‌ای که روی فایل‌های آن کار می‌کنیم
Repository همان پوشه + پوشه مخفی .git
git init تبدیل یک پوشه معمولی به Repository

مقدمه ای بر مفهوم

یک پوشه معمولی را در نظر بگیرید. قبل از اجرای git init، این پوشه فقط یک محیط کاری ساده است.

بعد از اجرای git init:

  • پوشه همچنان همان پوشه است
  • اما اکنون یک سیستم ثبت تاریخچه درون آن فعال شده است

پوشه + .git = Repository

بررسی وضعیت Repository

پس از اجرای git init می‌توانید دستور زیر را اجرا کنید:

git status

این دستور هیچ تغییری ایجاد نمی‌کند؛ فقط وضعیت فعلی پروژه را گزارش می‌دهد. معمولاً در این مرحله پیامی دریافت می‌کنید که نشان می‌دهد هنوز هیچ Commitی وجود ندارد.

نتیجه

با اجرای git init:

  • پوشه شما به Repository تبدیل می‌شود
  • ساختار داخلی Git ایجاد می‌شود
  • امکان ثبت تاریخچه فراهم می‌شود

اما هنوز هیچ فایلی در تاریخچه ثبت نشده است.
