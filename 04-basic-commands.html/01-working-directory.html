01 — Working Directory

تعریف

Working Directory دایرکتوری‌ای در فایل‌سیستم است که فایل‌های پروژه در آن قرار دارند و شما مستقیماً آن‌ها را ویرایش می‌کنید.

تا پیش از ایجاد Repository، این پوشه:

  • تحت کنترل Git نیست
  • هیچ تاریخچه‌ای ندارد
  • هیچ نسخه‌ای ذخیره نمی‌کند
  • هیچ فایلی را رهگیری نمی‌کند

در این مرحله ما صرفاً با فایل‌سیستم کار می‌کنیم.

موقعیت در معماری Git

Git بر کل سیستم شما نظارت ندارد.

مدل ذهنی Git شامل سه ناحیه است:

  1. Working Directory
  2. Staging Area
  3. Repository

Working Directory نقطه‌ی آغاز چرخه‌ی تغییرات است. هر تغییری ابتدا در این ناحیه اتفاق می‌افتد.

تا زمانی که Repository ایجاد نشود، این پوشه هیچ تفاوتی با سایر پوشه‌های سیستم ندارد.

دستورات پایه در Working Directory

مشاهده مسیر فعلی

pwd

نمایش مسیر دایرکتوری فعلی.

ایجاد Working Directory جدید

mkdir git-practice
cd git-practice
pwd

اکنون داخل Working Directory پروژه قرار دارید، اما هنوز Git فعال نشده است.

کار با فایل‌ها

echo "hello" > note.txt
echo "world" >> note.txt
ls

فایل تغییر می‌کند، اما:

  • ثبت نمی‌شود
  • نسخه‌سازی نمی‌شود
  • تاریخچه ایجاد نمی‌شود

چرا؟ چون هنوز Repository وجود ندارد.

وضعیت در این مرحله

مؤلفه وضعیت
فایل‌ها وجود دارند
ساختار پروژه وجود دارد
رهگیری تغییرات فعال نیست
نسخه‌بندی فعال نیست
Repository ایجاد نشده
