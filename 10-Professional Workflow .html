10 — Professional Workflow (End-to-End Scenario)

در این فصل یک سناریوی واقعی تیمی را از ابتدا تا انتشار نسخه نهایی اجرا می‌کنیم.

هدف: اتصال تمام مفاهیم دوره در یک جریان عملی.

سناریو

یک تیم سه‌نفره در حال توسعه یک Feature جدید برای پروژه است.

پروژه دارای:

  • Branch اصلی: main
  • نسخه پایدار فعلی: v1.2.0

هدف:
افزودن سیستم احراز هویت جدید.

مرحله 1 — همگام‌سازی با تیم

ابتدا آخرین تغییرات را دریافت می‌کنیم:

git checkout main
git pull origin main

همیشه قبل از شروع کار، Branch اصلی باید به‌روز باشد.

مرحله 2 — ساخت Feature Branch

git switch -c feature/auth-system

اصل حرفه‌ای:

یک Branch = یک هدف مشخص

مرحله 3 — توسعه و Commitهای هدفمند

در طول توسعه:

. git add 
" git commit -m "Add authentication service structure

"git commit -m "Implement JWT validation

Commitها باید:

  • کوچک
  • مشخص
  • معنادار

باشند.

مرحله 4 — Push و ایجاد Pull Request

git push -u origin feature/auth-system

سپس در پلتفرم میزبانی:

  • ایجاد Pull Request
  • درخواست Code Review
  • بررسی CI/CD

مرحله 5 — به‌روزرسانی Branch در صورت تغییر main

اگر در این فاصله main تغییر کرده باشد:

git fetch origin
git rebase origin/main

یا:

git merge origin/main

تیم باید یک استراتژی ثابت داشته باشد:

Rebase-based یا Merge-based

Consistency مهم‌تر از انتخاب استراتژی است.

مرحله 6 — Merge به main

پس از تأیید:

  • Merge (معمولاً با Squash یا --no-ff)
  • حذف Feature Branch
git branch -d feature/auth-system
git push origin --delete feature/auth-system

مرحله 7 — انتشار نسخه جدید

پس از تکمیل Featureها:

git checkout main
git tag v1.3.0
git push origin v1.3.0

Tag نشان‌دهنده یک نسخه پایدار است.

چرخه کامل حرفه‌ای

Pull
→ Branch
→ Develop
→ Commit
→ Push
→ Pull Request
→ Review
→ Merge
→ Tag
→ Release

این چرخه پایه‌ی توسعه حرفه‌ای در Git است.

اشتباهات رایج در تیم‌ها

  • Commitهای بزرگ و نامشخص
  • Branchهای طولانی‌مدت
  • Force Push روی Branch مشترک
  • Merge بدون Review
  • نداشتن Convention ثابت

مدل از مفهوم

Git یک گراف از Snapshotهاست.

Workflow حرفه‌ای یعنی:

کنترل حرکت در این گراف
به‌صورت هماهنگ، قابل ردیابی و پایدار.

پایان دوره

اکنون شما:

  • معماری Git را درک می‌کنید
  • Branch و Merge را حرفه‌ای مدیریت می‌کنید
  • Remote و همکاری تیمی را می‌شناسید
  • Workflow سازمانی را اجرا می‌کنید
  • نسخه منتشر می‌کنید

Git دیگر برای شما مجموعه‌ای از دستورات نیست.

یک سیستم کنترل تاریخچه قابل مهندسی است.
